# Phase 2 â€” UI Design Contract

> Visual and interaction contract for Phase 2: Observability Infrastructure. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification: Backend-only (data layer).** Phase 2 builds no frontend UI. It builds the Tauri commands, event bus, ring buffers, and stat methods that Phase 3 (Dashboard UI) will consume. This contract documents: (1) the existing design system baseline that all future phases inherit, (2) the IPC data shape contract that Phase 3 will render, and (3) copywriting strings embedded in backend response structs (actionable hints, status labels, error messages).

## Copywriting Contract

Phase 2 embeds user-facing copy in backend response structs. Phase 3 renders these strings directly. All copy documented here is the source of truth.

### Sync State Labels (DriveStatus.sync_state)

| State Value | Display Label | When |
|-------------|--------------|------|
| `"up_to_date"` | Up to date | Last delta sync completed with no errors |
| `"syncing"` | Syncing | Delta sync or writeback in progress |
| `"error"` | Sync error | Last delta sync or writeback failed |

### Online/Offline Labels (DriveStatus.online)

| State | Display Label | When |
|-------|--------------|------|
| `true` | Online | Network reachable, Graph API responding |
| `false` | Offline | Network unreachable or VFS timeout triggered offline flag |

### Activity Type Labels (ObsEvent.Activity.activity_type)

| Type Value | Display Label | When |
|------------|--------------|------|
| `"synced"` | Synced | File downloaded from server via delta sync |
| `"uploaded"` | Uploaded | User-initiated write completed upload |
| `"deleted"` | Deleted | File deleted (local or remote) |
| `"conflict"` | Conflict | Conflict copy created |

### Pin Health Labels (PinHealthInfo.status)

| Status Value | Display Label | When |
|-------------|--------------|------|
| `"downloaded"` | Downloaded | All files present in disk cache, no server changes since last pin sync |
| `"partial"` | Partial | Not all files in pinned directory tree are present in disk cache |
| `"stale"` | Stale | Server content changed since pin was last synced |

### Error Action Hints (DashboardError.action_hint)

Each error entry carries a short actionable string. These are embedded in the backend:

| Error Type | Action Hint |
|------------|-------------|
| Auth 401 / token expired | Re-authenticate in Carmine settings |
| Permission denied (403) | Check your permissions for this drive |
| Resource not found (404) | This file or folder may have been deleted |
| Rate limited (429) | Too many requests -- sync will retry automatically |
| Server error (5xx) | Microsoft service issue -- sync will retry automatically |
| Network / timeout | Check your internet connection |
| File locked | File is locked by another user -- try again later |
| Conflict detected | A conflict copy was created in the same folder |
| Writeback failed | Upload failed -- file queued for retry |
| Upload failed | Upload failed -- check file permissions and size |
| Drive deleted | This drive was deleted or access was revoked |

### Empty States

| Context | Heading | Body |
|---------|---------|------|
| No errors | No recent errors | Everything is running smoothly. Errors will appear here when they occur. |
| No activity | No recent activity | File sync activity will appear here once drives are mounted. |
| No pins | No offline pins | Pin a folder for offline access from your file manager. |
| No mounts | No drives mounted | Add a drive in the Mounts panel to get started. |

### Auth Degradation Banner

| State | Copy |
|-------|------|
| `auth_degraded: true` | Sign-in expired. Re-authenticate to resume sync. |
| `auth_degraded: false` | (banner hidden) |

### Destructive Actions

Phase 2 introduces no new destructive actions. Existing destructive actions (unmount, clear cache, sign out, unpin) are unchanged.

## IPC Data Shape Contract

Phase 2 defines the JSON response shapes that Phase 3 will consume. These are the Tauri command return types serialized as JSON.

### get_dashboard_status

```json
{
  "drives": [
    {
      "driveId": "string",
      "name": "string",
      "mountPoint": "string",
      "online": true,
      "lastSynced": "2026-03-18T12:00:00Z",
      "syncState": "up_to_date",
      "uploadQueue": {
        "queueDepth": 0,
        "inFlight": 0,
        "failedCount": 0,
        "totalUploaded": 0,
        "totalFailed": 0
      }
    }
  ],
  "authenticated": true,
  "authDegraded": false
}
```

### get_recent_errors

```json
[
  {
    "driveId": "string",
    "fileName": "Q4-Report.xlsx",
    "remotePath": "/Documents/Reports/Q4-Report.xlsx",
```
