# Phase 4 -- UI Design Contract

> Visual and interaction contract for the Visual Polish & Hardening phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase is a refinement pass on existing UI surfaces (settings.html, wizard.html). No new panels, no new data sources. All changes cascade through the existing CSS custom property token system in `:root`.

## Spacing Scale

All values are multiples of 4. The existing token names (`--space-1` through `--space-8`) are preserved. Values are mapped to a consistent 4-based scale, replacing the current rem-based tokens.

| Token | Current Value | New Value | Usage |
|-------|---------------|-----------|-------|
| `--space-1` | 0.25rem (4px) | 4px | Icon gaps, inline padding, tight internal gaps |
| `--space-2` | 0.5rem (8px) | 8px | Compact element spacing, button icon gap |
| `--space-3` | 0.75rem (12px) | 12px | Nav item padding, card internal padding |
| `--space-4` | 1rem (16px) | 16px | Default section padding, card gaps, status bar padding |
| `--space-5` | 1.25rem (20px) | 20px | Section heading margin-bottom |
| `--space-6` | 1.5rem (24px) | 24px | Main content horizontal padding, major section gaps |
| `--space-8` | 2rem (32px) | 32px | Main content vertical padding, page-level spacing |

Exceptions:
- Touch targets on icon buttons: minimum 28px hit area (7x the base unit) for comfortable click targets on desktop. This is below the 44px mobile guideline but appropriate for a desktop-only app.
- Main content padding changes from `22px 28px` to `--space-8 --space-6` (32px 24px) for more breathing room.

## Typography

Font: Inter Variable, weight range 100-900.

The existing type scale uses too many fractional pixel sizes (11px, 11.5px, 12px, 12.5px, 13px, 16px, 18px). This contract consolidates to 4 distinct sizes with 3 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 | 1.5 | Default text, setting labels `.label-text`, activity names, error messages, cache text, mount names, pin names |
| Small / Meta | 11px | 400 | 1.4 | Timestamps, sublabels `.label-sub`, error hints, mount paths, pin meta, health badge text, activity tags, section heading overline |
| Label / UI | 13px | 500 | 1.4 | Button text, nav items, card names `.drive-card-name`, error file names, handler extensions, sidebar title |
| Heading | 16px | 600 | 1.25 | Step titles `h1`, panel headings (if added). Wizard large title: 18px weight 600 (existing `step-title-lg`). |

Weight system:
- 400 (regular): Body text, descriptions, metadata
- 500 (medium): Interactive labels, buttons, nav items, card titles
- 600 (semibold): Section headings (overline), heading text, active emphasis

Migration from current values:
- 12.5px button text --> 13px weight 500
- 11.5px inputs --> 12px weight 400 (exception: inputs get 12px for optical clarity at small size, between the two body tiers)
- 12.5px nav items --> 13px weight 500
- 11px section headings --> 11px weight 600 (unchanged)
- 13px label text --> 13px weight 400 (unchanged)

Input exception: `input`, `select`, `.input` elements use 12px at weight 400 with line-height 1.4. This is a deliberate half-step between the 11px meta tier and 13px body tier, providing optical balance inside compact form controls.

## Color

Direction: **Soft dark refresh** -- lift the dark palette for more visual warmth and less cave-like heaviness. Inspired by Linear/Raycast soft dark, not stark terminal-black. `color-scheme: dark` is preserved.

### Background Tokens (60% Dominant)

| Token | Current | New | Rationale |
|-------|---------|-----|-----------|
| `--bg-base` | `#1c1d27` | `#1e2030` | Lifted from near-black. Warmer blue undertone. Still unmistakably dark. |
| `--bg-surface` | `#20222e` | `#242637` | Sidebar, secondary surfaces. Slightly lifted. |
| `--bg-elevated` | `#252733` | `#2a2d3e` | Cards, inputs, drive cards, error entries. More visible lift from surface. |

### Text Tokens

| Token | Current | New | Rationale |
|-------|---------|-----|-----------|
| `--text-primary` | `#e8e9f0` | `#e2e4f0` | Slightly softer white. Retains high readability. |
| `--text-secondary` | `#8c90aa` | `#8e92b0` | Minor warmth adjustment. Used for status text, descriptions. |
| `--text-muted` | `#5c6080` | `#6e7298` | Significant lift for better readability. Current value fails WCAG AA against dark backgrounds. New value reaches ~4.5:1 contrast ratio against `--bg-base`. |

### Border Tokens (30% Secondary)

| Token | Current | New | Rationale |
|-------|---------|-----|-----------|
| `--border` | `rgba(255,255,255,0.09)` | `rgba(255,255,255,0.08)` | Slightly softer. Used for sidebar border, card borders. |
| `--border-row` | `rgba(255,255,255,0.07)` | `rgba(255,255,255,0.05)` | More subtle row separators. Background contrast between base and elevated now carries more of the visual separation. |

### Accent Token (10%)

| Token | Current | New | Rationale |
|-------|---------|-----|-----------|
| `--accent` | `#99222E` | `#99222E` | **Unchanged.** Carmine red is brand identity. |
| `--accent-hover` | `#b52a38` | `#b52a38` | **Unchanged.** |
| `--accent-bg` | `rgba(153,34,46,0.85)` | `rgba(153,34,46,0.80)` | Slightly reduced opacity for better integration against lifted backgrounds. |
| `--accent-glow` | `0 0 20px rgba(153,34,46,0.2)` | `0 0 16px rgba(153,34,46,0.15)` | Tighter, subtler glow. |

Accent reserved for:
- Primary action buttons (accent-bg background)
- Active sidebar nav item background
- Active toggle switch track
- Focus-visible outline ring
- Sign Out link color
- Checkbox/radio selected state
- Links in attribution text
- Sidebar logo gradient

NOT for: body text, section headings, card borders, status indicators. Status uses semantic colors only.

### Semantic Colors

| Token | Current | New | Usage |
|-------|---------|-----|-------|
| `--success` | `#22c55e` | `#34d399` | Slightly warmer green. Status dot (ok), cache bar (low), health badge (downloaded), toggle checked state in wizard. |
| `--danger` | `#ef4444` | `#f87171` | Slightly softer red. Error indicators, destructive action hover, cache bar (high), health badge (stale). |
| `--warning` | `#f59e0b` | `#fbbf24` | Slightly brighter amber. Syncing status dot, conflict borders, cache bar (medium), health badge (partial), auth banner. |

### Shadow & Elevation

| Token | Current | New | Usage |
|-------|---------|-----|-------|
| `--shadow-sm` | `0 1px 3px rgba(0,0,0,0.4)` | `0 1px 2px rgba(0,0,0,0.25)` | Subtle depth on small elevated elements. |
| `--shadow-md` | `0 4px 16px rgba(0,0,0,0.5)` | `0 4px 12px rgba(0,0,0,0.35)` | Drive cards, error entries. Less heavy. |
| `--shadow-glow` | `var(--accent-glow)` | `var(--accent-glow)` | Unchanged alias. |

## Border Radius Scale

Direction: Rounder corners for a more modern, softer feel.

| Token | Current | New | Usage |
|-------|---------|-----|-------|
| `--radius-sm` | 4px | 6px | Error entry, small badges, activity tags |
| `--radius-md` | 6px | 8px | Buttons, inputs, nav items, drive cards, sidebar logo |
| `--radius-lg` | 10px | 12px | Modal-like containers (if any), large panels |

The toggle switch track border-radius stays at 8px (half of 16px height) for pill shape.
