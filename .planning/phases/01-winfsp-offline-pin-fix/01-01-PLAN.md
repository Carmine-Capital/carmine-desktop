<objective>
Add VFS-path timeouts to all Graph API calls made from VFS callbacks, and fix unbounded log growth.

Purpose: Prevent Explorer hangs by ensuring no VFS callback blocks longer than 5 seconds on a network call. The first timeout transitions the VFS to offline mode, protecting all subsequent calls. Log rotation prevents disk exhaustion over weeks of operation.

Output: Modified `client.rs` (timeout-capable client), modified `core_ops.rs` (timeout-wrapped VFS-path calls), modified `main.rs` (log rotation).
</objective>

Create a private helper method on `CoreOps`:

```rust
/// Execute a Graph API future with a VFS-path timeout.
/// If the future doesn't complete within `VFS_GRAPH_TIMEOUT`, sets the
/// offline flag and returns `VfsError::TimedOut`.
fn graph_with_timeout<T>(&self, fut: impl std::future::Future<Output = carminedesktop_core::Result<T>>) -> VfsResult<T> {
    match self.rt.block_on(tokio::time::timeout(VFS_GRAPH_TIMEOUT, fut)) {
        Ok(Ok(val)) => Ok(val),
        Ok(Err(e)) => {
            if matches!(&e, carminedesktop_core::Error::Network(_)) {
                self.set_offline();
            }
            Err(VfsError::from_core_error(e))
        }
        Err(_elapsed) => {
            self.set_offline();
            Err(VfsError::TimedOut)
        }
    }
}
```
