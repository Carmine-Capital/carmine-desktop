**Fix 2: removeSource() missing success feedback (wizard.js)**

In wizard.js, the `removeSource()` function at ~line 564-579 has no success feedback. After the successful removal (after line 578 `updateGetStartedBtn();`), add:

```javascript
showStatus('Source removed', 'success');
```

The function already calls `showStatus(formatError(e), 'error')` on failure (line 572), so the success path just needs the matching feedback. The final function should look like:

```javascript
async function removeSource(mountId) {
  const btn = document.querySelector('[data-action="remove-source"][data-mount-id="' + CSS.escape(mountId) + '"]');
  if (btn) btn.disabled = true;
  if (mountId) {
    try {
      await invoke('remove_mount', { id: mountId });
    } catch (e) {
      if (btn) btn.disabled = false;
      showStatus(formatError(e), 'error');
      return;
    }
  }
  state.addedSources = state.addedSources.filter(s => s.mountId !== mountId);
  renderAddedSources();
  updateGetStartedBtn();
  showStatus('Source removed', 'success');
}
```

`showStatus` is already available in wizard.js scope (imported from ui.js, used on line 130, 167, 194, 251, 253, etc.).
