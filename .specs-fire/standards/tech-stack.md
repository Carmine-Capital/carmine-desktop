# Tech Stack

## Overview

CloudMount is a cross-platform desktop application written in Rust that mounts Microsoft OneDrive and SharePoint document libraries as local filesystems. It uses a 6-crate Cargo workspace with platform-specific VFS backends (FUSE on Linux/macOS, Cloud Files API on Windows) and a Tauri v2 system tray UI.

### Linux Development Environment (Immutable Distros)

**Tool**: `toolbox` (Podman-based — ships with Fedora Silverblue, Kinoite, and other immutable distros)
**Container name**: `cloudmount-build`

Developers on immutable Linux distributions (Fedora Silverblue, Kinoite, etc.) **must** use the `cloudmount-build` toolbox container for all Rust/build/test operations. Do not install Rust or build tools on the host system.

#### Creating the container

```bash
# Create the container (Fedora base image)
toolbox create --container cloudmount-build

# Install system build dependencies
toolbox run --container cloudmount-build sudo dnf install -y \
  gcc gcc-c++ pkg-config \
  openssl-devel \
  fuse3-devel \
  dbus-devel

# Install Tauri desktop dependencies (only needed for --features desktop)
toolbox run --container cloudmount-build sudo dnf install -y \
  webkit2gtk4.1-devel \
  libappindicator-gtk3-devel \
  librsvg2-devel

# Install Rust toolchain via rustup
toolbox run --container cloudmount-build bash -c \
  'curl --proto '"'"'=https'"'"' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y'
```

#### Running commands inside the container

```bash
# Enter an interactive shell
toolbox enter cloudmount-build

# Or prefix individual commands
toolbox run --container cloudmount-build cargo build --all-targets
toolbox run --container cloudmount-build cargo test --all-targets
toolbox run --container cloudmount-build cargo clippy --all-targets --all-features
```

> The project root is automatically bind-mounted inside the toolbox — no path changes needed.

## Key Dependencies
