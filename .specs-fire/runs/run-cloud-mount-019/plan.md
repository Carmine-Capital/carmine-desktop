## Approach

Fix 5 tray/notification issues: dead menu item handler, misleading notification text, CSP-violating `win.eval()` calls, silent update check failure, and undocumented Linux tray limitation.

### 1. Remove dead "Open Mount Folder" from initial static menu (`tray.rs:12`)

The `setup()` function creates an "Open Mount Folder" menu item with id `"open_folder"`, but `handle_menu_event` has no case for it (falls through to `_ => {}`). Since `update_tray_menu()` replaces the entire menu with per-mount items anyway, the cleanest fix is to remove the dead item from the initial static menu. The static menu is only shown briefly before `update_tray_menu()` runs.

### 2. Fix auth_expired notification text (`notify.rs:65-70`)

The notification says "Click to re-authenticate" but has no click handler. Tauri v2's notification plugin supports `on_action`, but it requires additional plugin configuration and action type registration. The simpler fix: remove "Click to" from the text, making it "Sign-in expired. Open CloudMount to re-authenticate." This is accurate since left-clicking the tray opens the wizard when unauthenticated.

### 3. Replace `win.eval()` with Tauri events (`tray.rs:138,156`)

Two `win.eval()` calls bypass CSP:
- `win.eval("goToAddMount()")` in `open_or_focus_wizard` (line 138)
- `win.eval("loadSettings(); loadMounts();")` in `open_or_focus_window` (line 156)

Fix: Emit Tauri events from Rust, listen in JS.
- Rust: `app.emit("navigate-add-mount", ())` and `app.emit("refresh-settings", ())`
- wizard.js: `listen("navigate-add-mount", () => goToAddMount())`
- settings.js: `listen("refresh-settings", () => { loadSettings(); loadMounts(); })`

### 4. Add notification for manual update check failure (`update.rs:144-145`)

`handle_manual_check` only logs `tracing::warn` on error. Fix: add `notify::update_check_failed()` helper and call it.

### 5. Document Linux tray left-click limitation (`tray.rs:35`)

Add a code comment explaining that AppIndicator backend on Linux may not fire `TrayIconEvent::Click`, and all functionality is accessible via right-click menu.
