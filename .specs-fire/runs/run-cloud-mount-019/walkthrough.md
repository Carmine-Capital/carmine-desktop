# Walkthrough: fix-tray-notifications

**Run**: run-cloud-mount-019
**Completed**: 2026-03-09

## Summary

Fixed 5 tray/notification issues: removed dead menu item, fixed misleading notification text, replaced CSP-violating `win.eval()` with Tauri events, added user-visible notification for update check failures, and documented Linux tray limitation.

## Changes

### 1. Removed dead "Open Mount Folder" static menu item

**File**: `crates/cloudmount-app/src/tray.rs`

The `setup()` function created an "Open Mount Folder" menu item (`open_folder`) that had no handler in `handle_menu_event` (fell through to `_ => {}`). Since `update_tray_menu()` immediately replaces the entire menu with per-mount folder items, this static item was dead code. Removed it from the initial menu.

### 2. Fixed auth_expired notification text

**File**: `crates/cloudmount-app/src/notify.rs`

Changed "Click to re-authenticate" to "Open CloudMount to re-authenticate". The notification had no click handler, and Tauri v2's notification action API would require additional plugin configuration. The new text is accurate — left-clicking the tray icon opens the wizard when unauthenticated.

### 3. Replaced `win.eval()` with Tauri events

**Files**: `tray.rs`, `wizard.js`, `settings.js`

Two `win.eval()` calls bypassed CSP:
- `win.eval("goToAddMount()")` → `win.emit("navigate-add-mount", ())`
- `win.eval("loadSettings(); loadMounts();")` → `win.emit("refresh-settings", ())`

Added `Emitter` trait import to `tray.rs`. Added `listen("navigate-add-mount", ...)` in `wizard.js` and `listen("refresh-settings", ...)` in `settings.js`. The `settings.js` file also needed the `listen` import from `window.__TAURI__.event`.

### 4. Added update check failure notification

**Files**: `notify.rs`, `update.rs`

Added `notify::update_check_failed()` helper following the existing `send()` pattern. Called it from `handle_manual_check()` in the `Err` branch, which previously only logged via `tracing::warn`.

### 5. Documented Linux tray left-click limitation

**File**: `tray.rs`

Added comment on `show_menu_on_left_click(false)` explaining that AppIndicator backend on Linux may not fire `TrayIconEvent::Click` for left-click, and that all functionality is accessible via the right-click menu.

## Decisions

| Decision | Rationale |
|----------|-----------|
| Remove static open_folder instead of adding handler | `update_tray_menu()` replaces the menu with per-mount items immediately; a generic "open first mount" handler would be confusing |
| Fix notification text instead of adding click handler | Tauri v2 notification actions require plugin setup and action type registration — disproportionate effort for this fix |
| Use `win.emit` not `app.emit` | Events emitted via `win.emit` are scoped to the specific webview window, matching the intent (wizard gets navigate event, settings gets refresh event) |

## Developer Notes

- The `Emitter` trait must be imported to use `win.emit()` on `WebviewWindow`. The compiler error message is helpful — it suggests the import.
- `win.emit` sends events to the specific webview. `app.emit` would send to all windows. Using `win.emit` is correct here since each event targets a specific window.
