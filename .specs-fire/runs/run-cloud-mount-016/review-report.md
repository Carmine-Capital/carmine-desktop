### 1. [Cross-platform] Colon in `local:*` pending file paths breaks Windows

- **File**: `crates/cloudmount-cache/src/writeback.rs:25-40`
- **Issue**: `pending_path()` used raw `item_id` as filename. For `local:{nanos}` IDs, the colon is illegal on Windows (NTFS). This would cause `persist()` to fail silently on Windows, defeating the crash-safety fix (Fix 5).
- **Fix**: Added `sanitize_filename()` (`:` → `%3A`) in `pending_path()` and `unsanitize_filename()` (`%3A` → `:`) in `list_pending()`. The encoding is unambiguous since `%` does not appear in Graph API item IDs.
- **Test added**: `test_writeback_local_colon_id_roundtrips` verifies write/list/read/remove with `local:` IDs.
