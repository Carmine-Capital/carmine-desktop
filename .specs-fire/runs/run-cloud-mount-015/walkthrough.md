## Key Implementation Details

### 1. CI Clippy All Platforms

The `Clippy (desktop)` step in `ci.yml` was gated with `if: runner.os == 'Linux'`, meaning Windows and macOS desktop code paths were never linted. Since all three matrix runners already have the required dependencies (Linux installs GTK/WebKit, macOS/Windows have native WebView), removing the gate is safe.

### 2. Workspace Dependency for libc

The `libc` crate was declared inline (`libc = "0.2"`) in `cloudmount-vfs/Cargo.toml` instead of through the workspace root. This violates the project convention that ALL deps go through `[workspace.dependencies]`.

### 3. parse_cache_size Cfg Gate

The function had `#[cfg(any(target_os = "linux", target_os = "macos", feature = "desktop"))]` — mixing platform and feature gates on a function that contains no platform-specific code. The callers themselves are properly gated, so the function is dead_code on Windows-without-desktop. Using `#[allow(dead_code)]` with a comment is the correct minimal fix.

### 4. Unix-Specific Test Gating

`test_expand_mount_point_home` uses hardcoded `/home/` paths and asserts `starts_with("/home/")` — this would be semantically wrong on Windows. Gating with `#[cfg(unix)]` keeps it out of Windows test runs.

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| parse_cache_size gate removal | `#[allow(dead_code)]` instead of removing entirely | Function has no callers on Windows-without-desktop; removing cfg entirely would trigger dead_code warning in CI |
| Test gating approach | `#[cfg(unix)]` vs platform-appropriate paths | Simpler — the test is inherently testing a Unix path pattern |
