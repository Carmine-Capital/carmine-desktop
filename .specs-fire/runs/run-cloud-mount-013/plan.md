## Approach

Fix six cross-platform issues in the `cloudmount-vfs` crate, each a targeted change:

### 1. Replace hardcoded errno values with libc constants (`mount.rs`)

**Current**: `raw == Some(107) || raw == Some(5)` — 107 is ENOTCONN on Linux but 57 on macOS.
**Fix**: Replace with `raw == Some(libc::ENOTCONN as i32) || raw == Some(libc::EIO as i32)`.
**Gate**: This function is already FUSE-only (Linux/macOS), `libc` is already a dep.

### 2. Case-insensitive `find_child` on Windows (`core_ops.rs`)

**Current**: All three comparison points use `item.name == name` (case-sensitive).
**Fix**: Add a `names_match(a, b)` helper that uses `eq_ignore_ascii_case` on Windows, exact match on other platforms. Apply at all three name-comparison sites in `find_child` (lines 365, 390) and the memory cache `children_map` lookup.
**Note**: Memory cache HashMap uses String keys — on Windows, need to compare by iterating rather than direct `.get()`.

### 3. CfApi `state_changed` — invalidate memory cache (`cfapi.rs`)

**Current**: Only logs `debug` and ignores pin/unpin state changes.
**Fix**: For each changed path, resolve the inode and invalidate the memory cache entry so the next access re-fetches from server.

### 4. CfApi `closed` — chunked reading for large files (`cfapi.rs`)

**Current**: `std::fs::read(&abs_path)` loads entire file into memory.
**Fix**: Below `SMALL_FILE_LIMIT` (4MB), keep current behavior. Above it, use chunked reading (BufReader with configurable chunk size) and stream directly to the writeback layer. Reuse `SMALL_FILE_LIMIT` from `cloudmount_graph`.

### 5. Move `libc` to workspace dependencies (`Cargo.toml`)

**Current**: `libc = "0.2"` declared inline in `crates/cloudmount-vfs/Cargo.toml`.
**Fix**: Add `libc = "0.2"` to `[workspace.dependencies]` in root `Cargo.toml`, change VFS to `libc = { workspace = true }`.

### 6. Descriptive RwLock panic messages (`inode.rs`)

**Current**: All `.unwrap()` on RwLock read/write guards.
**Fix**: Replace each `.unwrap()` with `.expect("inode table lock poisoned")` for clear diagnostics.
